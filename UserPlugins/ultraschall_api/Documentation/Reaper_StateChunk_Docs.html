QUEUED_RENDER_OUTFILE

Functioncall:

QUEUED_RENDER_OUTFILE string renderfilename_with_path_for_queue integer auto_increment string guid

Description:
Only in Render-queued-projects!

stores the output-filename with path for the render-queue. If omitted while project is in Resourcefolder/QueuedRenders, the renderfilename will be created like a regular projectfile using RENDERFILE and RENDERPATTERN

There can be more than one (i.e. with rendering stems). According to the settings in RENDER_STEMS, the first entry may refer to the render-targetfile of the master-track.

This is mostly for showing in the RenderQueue-window and for you to get, which files will be exported while queue-render. It has no other use.


Parameters:
 string renderfilename_with_path_for_queue the render-outputfilename, which will be created when the queued project is rendered  
 integer auto_increment 0, don't auto-increment renderfilename when renderfilename already exists; 16, autoincrement renderfilename, when renderfilename already exists  
 string guid a guid associated with this queue  

QUEUED_RENDER_ORIGINAL_FILENAME

Functioncall:

QUEUED_RENDER_ORIGINAL_FILENAME string projectfilename_with_path

Description:
Only in Render-queued-projects!

stores the original filename of the queued-project with path.
Omitted for projects, that are not saved yet but already in the render-queue.

Will be used by Reaper for the RENDER_PATTERN wildcard $project

It's folder will also be used, when neither RENDER_FILE or QUEUED_RENDER_OUTFILE are set as output files/folders.

Will be ignored, when rendering the project without the rendering queue.


Parameters:
 string projectfilename_with_path the original projectfilename including it's path
